Code Editors and IDE's

There are many code editors out there and many IDE's to choose from. Which one you choose will depend mostly on preference, but also on what you want to accomplish, whether its a native Windows App written in C#, or a simple website written in HTML CSS and JS. If you are looking at this then I am assuming that you are a newbie like me, so I will go over some basic text editors that I have used


Sublime Text editor

Sublime is an awesome text editor that is free (for the most part) and it is the one that I used to make this site and all my others. It is awesome for a few reasons:

  • It is very user friendly
  • It is very light weight and fast
  • It is free (not technically but you can ignore the prompt to purchase and it always works)
  • It is always being updated to support new languages
  • It is the same across all operating systems
It is not so awesome for a few reasons:
  • You will need to install a handful of plugins before you can benefit from ST’s full potential.
  • Git integration is somehow a bit shallow compared to other editors.
  • If you decide to purchase it, it is $70 which is a bit steep for an editor.
Download Sublime Here


Atom Text editor

Atom is another light-weight and fast text editor that I have seen many developers use when watching Youtube videos. It is awesome for these reasons:

  • It is completely free.
  • Very active community or users and lots of resources/tutorials on Github/Medium, etc.
  • The file-tree-view is really nice and perhaps the best one across all editors.
  • Lots of support from Github crew since they own it.
  • It is the same across all operating systems.
It is not so awesome for a few reasons:
  • Takes some time to get the right setup.
  • Has some issues handling large repos/folders and CPU usage can go pretty high. It’s a browser-based app (runs on Electron), and is a bit slow to load and sometimes to respond.
  • Most of the core is written in CoffeeScript, but the Github team is transitioning to ES6.
  • Strange syntax management sometimes, you get a different color for your variable name depending on what you typed before (const, let, var etc.).
Download Atom here


Brackets

Brackets is another really good and fast text editor. I have used it in the past but have since decided to go with Sublime. It is awesome for a few reasons:

  • It is Free!
  • It does Code completion out of the box.
  • The split window mode is really easy to use and intuitive.
  • When you mark any HTML element and press Cmd/Ctrl + E, Brackets opens all CSS associated with this element inline in the current window. From here, you can make any edits you need. Once you close it, any changes will automatically be applied to the stylesheet connected with this element. Nice!
It is not so awesome for a few reasons:
  • Needs plugins and packages to really be complete.
  • Supports only web languages out of the box.
  • Can also be slow at times but no big deal (especially compared to Atom).
Download Brackets here


Visual Studio Code

Visual studio code is a very popular text coding program especially in enterprise settings. The developers where I work use it to write their .NET and C# applications. I used it once or twice to try and get a node server going using npm. It is awesome for a few reasons:

  • It is stable.
  • It is fast.
  • It is reliable.
  • It has a debugger built in.
It is not so awesome for a few reasons:
  • The interface is not as polished as some of the other text editors.
  • The navigation system is a bit odd.
  • You can only get it on Microsoft.
Download Visual Studio here

Laying out Content

There are several different ways to lay out your content in the web browser. As a newbie, I can tell you that the sheer amount of difference in opinions of which to use can sometimes be overwhelming. I have personally used all three at least once, so I will list some pros and cons of each so that you can make your own decision on which to use.


Manual Layouts

One way of laying out a site is by using margins and relative positioning. This is pretty much what I did for this site, just to see if I could make something without Bootstrap like I always do. You have to manually set the margins, widths and heights, then you have to do some extensive media queries to get it all to work on mobile. For example, to make the sidebar on this site, I did the following CSS for an id of Navbar: #Navbar {top: 0; left: 0; height 100%; width: 200px;} Looks simple right! Well here is the thing, how are you gonna get the rest of the content to the right? well, if your viewport is fully expanded, you could simply add a margin-left: 200px so that the content starts 200px from the left of the screen leaving space for the sidebar. The main pro about this method is that it will work with pretty much all browsers, since it is just plain CSS.


Bootstrap 4

Bootstrap is a full framework which provides pre-defined classes, which makes it easy to create a layout and make it all responsive. I personally prefer Bootstrap and have used it on all my projects except this one. Basically, to use Bootstrap, you just link to the CDN (or download it once you get to backend stuff) then start using its layout system and predefined classes. One of the coolest parts about Bootstrap is the layout system. In the layout system you have containers, rows and colums. If you wanted to make three cards in line, for example, you would just make a div class of "container" (or container-fluid for full width) then a div class of row, then 3 columns of col-sm-3. There are 12 columns in Bootstrap, so if you want two side boxes that are equal, you can use col-sm-6 twice. I know this is an extreme simplification of it. To learn more visit the Bootstrap website and learn more!

Adding CSS

CSS stands for Cascading Style Sheets CSS describes how HTML elements are to be displayed on screen, paper, or in other media CSS saves a lot of work. It can control the layout of multiple web pages all at once External stylesheets are stored in CSS files. Without CSS, websites would look like the first HTML website ever made: CSS adds color, styling and more to your page, and is what makes websites look so good. For example, if you were to just add an image from a website using img src="image.google.com" it would default to whatever the image size is from the source. To fix this, you would go into your CSS document and select the Img tag, then change the properties like so: img {width: 450px; height: auto;}. The following code would make all images have a width of 450px and the height would be calculated depending on its aspect ration. If you wanted to control each image separately, you would have to add a class to the image such as img class="image1" then add the CSS: .image1 {width: 450px; height: auto;} To get started with CSS, head over to W3 Schools to learn more!
